# PRD: 내 강의실 MVP (PDF 자료 전용)

## 1. 문서 목적
- 본 문서는 내 강의실 페이지, 업로드 팝업, 보드 상세 페이지의 기능 요구사항과 UI 상호작용을 정의한다.
- 기술 아키텍처 원칙은 `docs/context.md`를 기준으로 하며, 본 문서는 제품 동작/화면 명세를 다룬다.

## 2. 범위
- 포함
- 내 강의실(목록/탭/폴더/상태 표시)
- 업로드 팝업(입력/파일 드롭/검증/생성 트리거)
- 보드 상세(미디어/PDF 뷰어, 스크립트, 플레이어 컨트롤)
- 제외
- 코드 구현 세부, 인프라 배포, 인증 고도화, 실시간 SSE/WebSocket

## 2.1 Stitch 시안 반영 원칙
- `screens/*/code.html`은 UI 레이아웃 레퍼런스로 사용한다.
- 도메인/업로드 정책 충돌 시 본 PRD와 `docs/domain-model.md`, `docs/api-contract.md`를 우선한다.
- 충돌 항목은 디자인 수정 TODO로 관리하고, 구현은 정책 기준으로 진행한다.

## 3. 용어 정의
- Folder: 강의명 단위 컨테이너
- Board: 강의 주제 단위 리소스
- Processing: 스크립트 추출/후보정이 진행 중인 상태

## 4. 화면 1: 내 강의실 페이지
### 4.1 좌측 사이드바
- 탭 관리
- 전체 보드 보기
- 즐겨찾기(Starred) 탭은 시안에 존재하나 MVP 기본 범위에서는 제외
- 폴더별 보기
- 현재 탭은 배경색으로 포커스 표시
- 상단에 사이드바 접기/펼치기 아이콘 버튼 제공
- 폴더 추가
- 폴더 추가 아이콘 클릭 시 폴더 목록 최상단에 인풋 박스 노출
- 입력 확정(Enter 또는 포커스 아웃) 시 폴더 생성 시도
- 빈 문자열, 중복명 정책 위반 시 인라인 에러 표시

### 4.2 중앙 메인 헤더
- 현재 탭 제목 표시
- 보드 개수 표시
- 우측에 보드 업로드 버튼 표시

### 4.3 보드 목록(표 형태)
- 컬럼
- 보드 제목(강의 주제)
- 음성 길이
- 생성일시
- 액션(더보기)
- hover 중인 행은 배경색 강조
- 각 행 우측에 더보기(`...`) 버튼 노출
- 더보기 메뉴
- 이름 변경
- 폴더 이동하기
- 삭제하기
- Processing 항목 규칙
- 회색 텍스트
- 클릭 불가
- 진행 상황 프로그레스 바 표시
- 보드 제목 클릭 시 보드 상세 페이지 이동

### 4.4 초기 진입 상태
- 내 강의실 첫 진입 시 `전체 보드` 탭이 기본 활성화

## 5. 화면 2: 업로드 팝업
### 5.1 열기/닫기
- 업로드 버튼 클릭 시 팝업 오픈
- 닫기 방식
- 우측 상단 `x` 버튼
- 팝업 외부 클릭
- `Esc` 키

### 5.2 입력 폼
- 강의명(필수)
- Folder 명으로 사용
- 동일 Folder가 없으면 생성
- 강의 주제(필수)
- Board 명으로 사용
- 강의 미디어 파일(필수)
- 지원: 비디오/오디오
- 드래그 앤 드롭 지원
- 강의 자료 파일(선택)
- 지원: PDF만 허용
- 드래그 앤 드롭 지원
- 파일 선택은 단일 파일만 허용

### 5.4 Stitch 시안 대비 수정 사항(필수)
- 자료 파일 안내 문구 `PDF, PPT, DOCX 지원`을 `PDF만 지원`으로 변경해야 한다.
- 자료 파일 input의 `multiple` 동작은 제거해야 한다.
- 미디어 최대 용량 문구(현재 시안 500MB)는 최종 정책 값과 일치하도록 교체해야 한다.

### 5.3 추출 버튼 클릭 시 동작
- 유효성 검증 통과 시 업로드 요청 생성
- 팝업 닫힘 후 내 강의실 페이지로 이동
- 목록 최상단에 Processing 항목 즉시 표시(낙관적 UI 또는 API 응답 기반)

## 6. 화면 3: 보드 상세 페이지
### 6.1 헤더
- 뒤로가기 아이콘 버튼(내 강의실로 이동)
- 강의명(Folder)
- 강의 주제(Board)

### 6.2 레이아웃
- 기본 7:3 분할(좌:우)
- 사용자 드래그로 비율 조절 가능
- 검증된 분할 라이브러리 사용 권장(예: react-resizable-panels)

### 6.3 좌측 메인
- 미디어가 영상이면 영상 플레이어 표시
- 오디오인 경우 PDF 뷰어 표시
- 하단에 자막 표시(음성 타임스탬프 동기화)
- PDF는 타임스탬프별 페이지 매핑을 강제하지 않음
- 사용자가 직접 스크롤 탐색 가능해야 함
- 참고: Stitch 시안의 고정 배경 이미지는 실제 구현에서 업로드 자산 렌더링으로 대체한다.

### 6.4 우측 스크립트 사이드바
- 상단
- 사이드바 접기 아이콘
- 검색 바
- 타임스탬프 구간별 스크립트 목록 표시
- 현재 시간대 구간은 테두리 + 배경색 강조
- 구간 클릭 시 해당 시간으로 seek
- 텍스트 더블 클릭 시 인라인 수정 가능

### 6.5 하단 플레이어 컨트롤
- 진행 바 표시
- 현재 시간/총 길이 표시
- 중앙: -5초, 재생/일시정지, +5초
- 우측: 배속, 볼륨, 전체화면
- 키보드
- 좌/우: -5초/+5초
- 상/하: 배속 감소/증가

## 7. 업로드 처리 기능 요구
- PDF가 업로드되면 PyMuPDF로 텍스트 추출
- 추출 텍스트로 LLM 키워드 추출
- Whisper 초안 생성 시 강의명, 강의 주제, 키워드(존재 시) 주입
- LLM 후보정 시 타임스탬프 불변 규칙 필수

## 8. 상태/에러 UX
- 상태 단계(권장): queued, processing, postprocessing, completed, failed
- 실패 시 사용자에게 간단 사유 표시(예: 파일 파싱 실패, STT 실패)
- Processing 중 항목은 더보기 메뉴에서 `삭제`만 허용한다.
- 실패 상태 항목의 액션 버튼 라벨은 `재시도`로 통일한다.

## 9. 수용 기준(AC)
- AC-1: 첫 진입 시 전체 보드 탭이 활성화된다.
- AC-2: 폴더 추가 버튼 클릭 시 최상단 인풋이 나타난다.
- AC-3: Processing 보드는 회색 + 클릭 불가 + progress bar를 만족한다.
- AC-4: 업로드 팝업은 `x/외부 클릭/Esc`로 닫힌다.
- AC-5: 자료 파일은 PDF만 허용되고, 비허용 형식은 즉시 검증 에러가 난다.
- AC-6: 상세 페이지는 7:3 기본 분할 및 드래그 리사이즈가 동작한다.
- AC-7: 스크립트 구간 클릭 시 해당 타임으로 이동한다.
- AC-8: 좌/우/상/하 키 조작이 플레이어에 반영된다.
- AC-9: 실패 상태 항목의 액션 버튼은 `재시도`로 표시된다.
- AC-10: 스크립트 편집은 저장 버튼 클릭 시에만 반영된다.

## 10. 확정 정책
- 동일 사용자 내 폴더명 중복은 허용하지 않는다.
- Processing 중 더보기 메뉴는 삭제만 허용한다.
- 실패 상태 액션 버튼 라벨은 `재시도`로 통일한다.
- 스크립트 편집은 저장 버튼 클릭 시 반영한다.
- 상세 페이지에서 오디오 + PDF 동시 레이아웃의 모바일 축소 규칙은 후속 스프린트에서 보완한다.

## 11. Stitch 검토 결과 요약
- 내 강의실
- 사이드바 접기 버튼, 폴더 추가 인풋, 행 hover, 더보기 메뉴는 요구사항과 정합하다.
- 업로드 팝업
- 강의명/세부 주제/드래그앤드롭 구조는 정합하다.
- 자료 파일 포맷 안내가 정책과 충돌(PDF 외 표기)하므로 수정 필요하다.
- 보드 상세
- 7:3 분할, 스크립트 하이라이트, 인라인 편집, 하단 플레이어 구조는 정합하다.
